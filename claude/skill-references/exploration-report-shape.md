# Exploration Report Shape

Report back with the following sections:

## Tech Stack
- Language, framework, build tool, key dependencies with versions

## Relevant Existing Patterns
- How similar features are currently implemented (file structure, naming, conventions)
- Specific files that serve as reference patterns for new work

## Integration Points
- How this repo connects to the systems involved in the feature
- Existing clients, adapters, or API functions that can be reused
- Authentication/authorization model

## Files to Modify vs Create
- Existing files that would need changes (with what kind of change)
- New files that would need to be created (with where they'd go)

## Open Questions
- Anything ambiguous, missing, or that needs a human decision
